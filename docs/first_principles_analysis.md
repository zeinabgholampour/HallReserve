# تحلیل پروژه بر اساس تفکر از اصول اولیه (first principles Thinking)

##  1.تعریف مسئله 
مدیریت وظایف در تیم های نرم افزاری اغلب با ابزار های سنگین یا غیر تخصصی انجام می شود.

## 2. تجزیه مسئله به اصول اولیه 
اصل1: هر تسک دارای مالک،وضعیت،ومهلت است.

اصل2:همکاری تیمی باید شفاف و قابل پیگیری باشد.

اصل3:کاربران باید در کمترین زمان به اطلاعات موردنیاز دسترسی یابند.

  ## 3.بازسازی راه حل از پایه
  - براساس اصول بالا،سامانه ای طراحی شد که تعامل برقرار می کند RESTful API با معماری:
  - رابط کاربری ساده و سریع.
  - تغییر وضعیت تسک ها به صورت بلادرنگ نمایش داده می شود.

   ## 4.فرضیات بررسی شده
  - نیاز به ثبت نام کاربران برای پیگیری وظایف.
  - استفاده از نقش ها (مدیر/عضو تیم)
  - تمرکز بر عملکرد به جای زیبایی بصری.

     ## 5.نتیجه گیری
    -تحلیل اصول اولیه منجر شد تا طراحی ساده تر،سریعتر و تمرکر بر منطق اصلی سیستم انجام گیرد.
